title: Uploading documents
description: Learn how to upload documents using presigned URLs

The Doc Optimizer API uses presigned S3 URLs for secure document uploads. This approach allows you to upload files directly to S3 without exposing AWS credentials.

## How it works

<Steps>
  <Step title="Request a presigned URL">
    Call the `/api/upload-url` endpoint with your document's content type and
    desired optimization level.
  </Step>
  <Step title="Upload your file">
    Use the presigned URL to upload your document directly to S3 via a PUT
    request.
  </Step>
  <Step title="Start processing">
    Use the returned `jobId` and `key` to initiate document processing.
  </Step>
</Steps>

## Request a presigned URL

Send a POST request to `/api/upload-url`:

<ParamField path="contentType" type="string" required>
  The MIME type of your document. Accepted values: - `application/pdf` -
  `application/vnd.openxmlformats-officedocument.wordprocessingml.document`
</ParamField>

<ParamField path="optimizationLevel" type="string" required>
  The compression level: `low`, `medium`, or `high`
</ParamField>

### Example request

```bash
curl -X POST https://781dzb51di.execute-api.us-east-1.amazonaws.com/api/upload-url \
  -H "Content-Type: application/json" \
  -H "x-api-key: your-api-key" \
  -d '{
    "contentType": "application/pdf",
    "optimizationLevel": "medium"
  }'
```

### Response

```json
{
  "success": true,
  "data": {
    "uploadUrl": "https://s3.amazonaws.com/bucket/uploads/original/abc-123.pdf?signature=...",
    "key": "uploads/original/abc-123.pdf",
    "jobId": "abc-123"
  },
  "timestamp": "2026-03-07T10:00:00.000Z"
}
```

<Warning>
  The presigned upload URL expires after **5 minutes (300 seconds)**. Upload
  your file immediately after receiving the URL.
</Warning>

<Warning>
  The `Content-Type` header on your PUT request must exactly match the
  `contentType` you provided when requesting the presigned URL. A mismatch will
  cause S3 to reject the upload.
</Warning>

## Upload your document

Use the `uploadUrl` from the response to PUT your file directly to S3:

```bash
curl -X PUT "https://s3.amazonaws.com/bucket/uploads/original/abc-123.pdf?signature=..." \
  -H "Content-Type: application/pdf" \
  --data-binary @document.pdf
```

A successful upload returns HTTP `200` with no body. Any non-200 response from S3 means the upload failed — re-request a presigned URL and try again.

## JavaScript example

```javascript
const API_BASE = "https://781dzb51di.execute-api.us-east-1.amazonaws.com/api";
const API_KEY = "your-api-key";

async function requestUploadUrl(file, optimizationLevel) {
  const response = await fetch(`${API_BASE}/upload-url`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": API_KEY,
    },
    body: JSON.stringify({
      contentType: file.type,
      optimizationLevel,
    }),
  });

  if (!response.ok) {
    const { error } = await response.json();
    throw new Error(error.message);
  }

  const { data } = await response.json();
  return data; // { uploadUrl, key, jobId }
}

async function uploadFileToS3(uploadUrl, file) {
  const response = await fetch(uploadUrl, {
    method: "PUT",
    headers: { "Content-Type": file.type },
    body: file,
  });

  if (!response.ok) {
    throw new Error("S3 upload failed. Request a new upload URL and retry.");
  }
}

// Usage
const file = document.getElementById("fileInput").files[0];
const { uploadUrl, key, jobId } = await requestUploadUrl(file, "medium");
await uploadFileToS3(uploadUrl, file);
console.log("Upload complete! Job ID:", jobId);
```

## Python example

```python
import requests

API_BASE = 'https://781dzb51di.execute-api.us-east-1.amazonaws.com/api'
API_KEY = 'your-api-key'

def request_upload_url(file_path, content_type, optimization_level):
    response = requests.post(
        f'{API_BASE}/upload-url',
        headers={'x-api-key': API_KEY},
        json={
            'contentType': content_type,
            'optimizationLevel': optimization_level,
        }
    )
    response.raise_for_status()
    return response.json()['data']  # { uploadUrl, key, jobId }

def upload_file_to_s3(upload_url, file_path, content_type):
    with open(file_path, 'rb') as f:
        response = requests.put(
            upload_url,
            headers={'Content-Type': content_type},
            data=f,
        )
    if not response.ok:
        raise Exception('S3 upload failed. Request a new upload URL and retry.')

# Usage
data = request_upload_url('document.pdf', 'application/pdf', 'medium')
upload_file_to_s3(data['uploadUrl'], 'document.pdf', 'application/pdf')
print(f"Upload complete! Job ID: {data['jobId']}")
```

## Error handling

<ResponseField name="400" type="Bad Request">
  Missing or invalid parameters:

```json
{
  "success": false,
  "error": {
    "message": "optimizationLevel must be low, medium, or high",
    "code": "INVALID_OPTIMIZATION_LEVEL"
  },
  "timestamp": "2026-03-07T10:00:00.000Z"
}
```

</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing API key in the `x-api-key` header.
</ResponseField>

<ResponseField name="403" type="Forbidden">
  Invalid, inactive, or expired API key.
</ResponseField>

<ResponseField name="500" type="Server Error">
  Failed to generate the upload URL. Retry after a short delay.
</ResponseField>
