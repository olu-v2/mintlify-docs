title: Processing documents
description: Learn how to start document processing with different optimization levels
---

Once you've uploaded a document to S3, call the `/api/process` endpoint to trigger optimization. Processing runs asynchronously — the endpoint returns immediately and the job runs in the background.

## Optimization levels

| Level    | iLovePDF mapping | Description                                               |
| -------- | ---------------- | --------------------------------------------------------- |
| `low`    | `low`            | Minimal compression, maximum quality preserved            |
| `medium` | `recommended`    | Balanced compression and quality. Best for most use cases |
| `high`   | `extreme`        | Maximum compression, smallest file size                   |

## Start processing

Send a POST request to `/api/process`:

<ParamField path="key" type="string" required>
  The S3 key returned from the `/api/upload-url` response (e.g.
  `uploads/original/abc-123.pdf`)
</ParamField>

<ParamField path="optimizationLevel" type="string" required>
  The optimization level: `low`, `medium`, or `high`
</ParamField>

<ParamField path="jobId" type="string" required>
  The job ID returned from the `/api/upload-url` response
</ParamField>

### Example request

```bash
curl -X POST https://781dzb51di.execute-api.us-east-1.amazonaws.com/api/process \
  -H "Content-Type: application/json" \
  -H "x-api-key: your-api-key" \
  -d '{
    "key": "uploads/original/abc-123.pdf",
    "optimizationLevel": "medium",
    "jobId": "abc-123"
  }'
```

### Response

```json
{
  "success": true,
  "data": {
    "message": "Processing started",
    "jobId": "abc-123"
  },
  "timestamp": "2026-03-07T10:00:00.000Z"
}
```

<Info>
  After a successful call, the job status transitions to `PENDING`, then
  `PROCESSING` as the Lambda begins working. Poll the `/api/status/:jobId`
  endpoint to track progress.
</Info>

<Warning>
  Always upload your file to S3 **before** calling `/api/process`. Processing
  will fail if the file does not exist at the specified `key`.
</Warning>

## JavaScript example

```javascript
const API_BASE = "https://781dzb51di.execute-api.us-east-1.amazonaws.com/api";
const API_KEY = "your-api-key";

async function startProcessing(key, optimizationLevel, jobId) {
  const response = await fetch(`${API_BASE}/process`, {
    method: "POST",
    headers: {
      "Content-Type": "application/json",
      "x-api-key": API_KEY,
    },
    body: JSON.stringify({ key, optimizationLevel, jobId }),
  });

  if (!response.ok) {
    const { error } = await response.json();
    throw new Error(error.message);
  }

  const { data } = await response.json();
  return data.jobId;
}

// Usage
const jobId = await startProcessing(
  "uploads/original/abc-123.pdf",
  "medium",
  "abc-123",
);
console.log("Processing started for job:", jobId);
```

## Python example

```python
import requests

API_BASE = 'https://781dzb51di.execute-api.us-east-1.amazonaws.com/api'
API_KEY = 'your-api-key'

def start_processing(key, optimization_level, job_id):
    response = requests.post(
        f'{API_BASE}/process',
        headers={'x-api-key': API_KEY},
        json={
            'key': key,
            'optimizationLevel': optimization_level,
            'jobId': job_id,
        }
    )
    response.raise_for_status()
    return response.json()['data']['jobId']

# Usage
job_id = start_processing('uploads/original/abc-123.pdf', 'medium', 'abc-123')
print(f'Processing started for job: {job_id}')
```

## Error handling

<ResponseField name="400" type="Bad Request">
  Missing or invalid parameters:

```json
{
  "success": false,
  "error": {
    "message": "Missing parameters: key, optimizationLevel, jobId",
    "code": "MISSING_PARAMETERS"
  },
  "timestamp": "2026-03-07T10:00:00.000Z"
}
```

</ResponseField>

<ResponseField name="401" type="Unauthorized">
  Missing API key in the `x-api-key` header.
</ResponseField>

<ResponseField name="403" type="Forbidden">
  Invalid, inactive, or expired API key.
</ResponseField>

<ResponseField name="500" type="Server Error">
  ```json
  {
    "success": false,
    "error": {
      "message": "Processing failed to start",
      "code": "PROCESSING_FAILED"
    },
    "timestamp": "2026-03-07T10:00:00.000Z"
  }
  ```
</ResponseField>
