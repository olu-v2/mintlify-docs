---
title: Downloading processed documents
description: Learn how to download your optimized documents after processing completes
---

Once your job status is `DONE`, call the `/api/download/:jobId` endpoint to get a presigned S3 URL and download your optimized file.

## Prerequisites

<Steps>
  <Step title="Verify job is DONE">
    Confirm the job status is `DONE` via the `/api/status/:jobId` endpoint
    before requesting a download URL.
  </Step>
  <Step title="Request a download URL">
    Call `/api/download/:jobId` to receive a presigned S3 URL.
  </Step>
  <Step title="Download the file">
    Use the presigned URL to fetch the optimized document directly from S3.
  </Step>
</Steps>

<Warning>
  The download endpoint returns a `400` error if the job status is not `DONE`.
  Always check the status first.
</Warning>

## Request a download URL

```bash
curl https://781dzb51di.execute-api.us-east-1.amazonaws.com/api/download/abc-123 \
  -H "x-api-key: your-api-key"
```

### Response

```json
{
  "downloadUrl": "https://s3.amazonaws.com/bucket/uploads/processed/abc-123_medium.pdf?signature=..."
}
```

<Info>
  The presigned download URL expires after **5 minutes (300 seconds)**. Download
  your file promptly — if the URL expires, call the endpoint again to get a
  fresh one.
</Info>

<Note>
  The download endpoint response is **not** wrapped in the standard `{ success, data }` envelope — `downloadUrl` is returned directly.
</Note>

## Download the file

Use the `downloadUrl` to fetch the optimized file from S3:

```bash
curl "https://s3.amazonaws.com/bucket/uploads/processed/abc-123_medium.pdf?signature=..." \
  -o optimized-document.pdf
```

## Output file naming

Processed files are stored in S3 with this naming convention:

```
uploads/processed/{jobId}_{optimizationLevel}.{ext}
```

For example, a PDF job with ID `abc-123` at `medium` compression:

```
uploads/processed/abc-123_medium.pdf
```

## Complete workflow

Here is the full upload → process → poll → download flow:

### JavaScript example

```javascript
const API_BASE = "https://781dzb51di.execute-api.us-east-1.amazonaws.com/api";
const API_KEY = "your-api-key";
const headers = { "x-api-key": API_KEY };

async function optimizeAndDownload(file, optimizationLevel = "medium") {
  // 1. Get presigned upload URL
  const urlRes = await fetch(`${API_BASE}/upload-url`, {
    method: "POST",
    headers: { ...headers, "Content-Type": "application/json" },
    body: JSON.stringify({ contentType: file.type, optimizationLevel }),
  });
  const { data: uploadData } = await urlRes.json();
  const { uploadUrl, key, jobId } = uploadData;

  // 2. Upload file directly to S3
  await fetch(uploadUrl, {
    method: "PUT",
    headers: { "Content-Type": file.type },
    body: file,
  });

  // 3. Start processing
  await fetch(`${API_BASE}/process`, {
    method: "POST",
    headers: { ...headers, "Content-Type": "application/json" },
    body: JSON.stringify({ key, optimizationLevel, jobId }),
  });

  // 4. Poll until DONE
  let job;
  while (true) {
    const statusRes = await fetch(`${API_BASE}/status/${jobId}`, { headers });
    job = await statusRes.json();

    if (job.status === "DONE") break;

    if (job.status === "FAILED") {
      throw new Error(`Processing failed: ${job.errorMsg}`);
    }

    await new Promise((resolve) => setTimeout(resolve, 2500));
  }

  // 5. Get download URL
  const dlRes = await fetch(`${API_BASE}/download/${jobId}`, { headers });
  const { downloadUrl } = await dlRes.json();

  // 6. Trigger browser download
  const a = document.createElement("a");
  a.href = downloadUrl;
  a.target = "_blank";
  a.rel = "noopener noreferrer";
  a.download = `optimized-${file.name}`;
  document.body.appendChild(a);
  a.click();
  document.body.removeChild(a);

  console.log("Download complete!");
}

// Usage
const file = document.getElementById("fileInput").files[0];
await optimizeAndDownload(file, "medium");
```

### Python example

```python
import requests
import time

API_BASE = 'https://781dzb51di.execute-api.us-east-1.amazonaws.com/api'
API_KEY = 'your-api-key'
headers = {'x-api-key': API_KEY}

def optimize_and_download(file_path, output_path, optimization_level='medium'):
    content_type = 'application/pdf'

    # 1. Get presigned upload URL
    url_res = requests.post(
        f'{API_BASE}/upload-url',
        headers=headers,
        json={'contentType': content_type, 'optimizationLevel': optimization_level}
    )
    url_res.raise_for_status()
    upload_data = url_res.json()['data']
    upload_url, key, job_id = upload_data['uploadUrl'], upload_data['key'], upload_data['jobId']

    # 2. Upload to S3
    with open(file_path, 'rb') as f:
        requests.put(upload_url, headers={'Content-Type': content_type}, data=f)

    # 3. Start processing
    requests.post(
        f'{API_BASE}/process',
        headers=headers,
        json={'key': key, 'optimizationLevel': optimization_level, 'jobId': job_id}
    )

    # 4. Poll until DONE
    while True:
        status_res = requests.get(f'{API_BASE}/status/{job_id}', headers=headers)
        job = status_res.json()

        if job['status'] == 'DONE':
            break

        if job['status'] == 'FAILED']:
            raise Exception(f"Processing failed: {job['errorMsg']}")

        time.sleep(2.5)

    # 5. Get download URL
    dl_res = requests.get(f'{API_BASE}/download/{job_id}', headers=headers)
    download_url = dl_res.json()['downloadUrl']

    # 6. Download file
    file_res = requests.get(download_url)
    with open(output_path, 'wb') as f:
        f.write(file_res.content)

    print(f'Saved to {output_path}')

# Usage
optimize_and_download('document.pdf', 'optimized-document.pdf', 'medium')
```

## Error handling

<ResponseField name="400" type="Bad Request">
  The job is not ready yet:

```json
{
  "error": "File is not ready yet",
  "status": "PROCESSING"
}
```

</ResponseField>

<ResponseField name="404" type="Not Found">
  The job ID does not exist:

```json
{
  "error": "Job not found"
}
```

</ResponseField>

<ResponseField name="500" type="Server Error">
  Failed to generate the download URL. Retry after a short delay.
</ResponseField>

## Best practices

<AccordionGroup>
  <Accordion title="Always check status before downloading">
    Calling the download endpoint before the job is `DONE` wastes an API request and returns a 400 error. Poll `/api/status/:jobId` first.
  </Accordion>

<Accordion title="Do not cache download URLs">
  Download URLs expire after 5 minutes. Always request a fresh URL rather than
  storing and reusing an old one.
</Accordion>

<Accordion title="Implement retry logic">
  Network issues can occur during S3 downloads. Use exponential backoff for
  production implementations.
</Accordion>

  <Accordion title="Job records expire after 48 hours">
    DynamoDB job records are automatically deleted 48 hours after creation. Download your file and store it before the record expires.
  </Accordion>
</AccordionGroup>
