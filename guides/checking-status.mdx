title: Checking job status
description: Learn how to monitor the status of your document processing jobs

After starting a processing job, poll the `/api/status/:jobId` endpoint to track progress. Once the status is `DONE`, you can request a download URL.

## Job statuses

Your job progresses through these statuses in order:

| Status       | Description                                          |
| ------------ | ---------------------------------------------------- |
| `CREATED`    | Job record created when the upload URL was generated |
| `PENDING`    | Processing request received, queued for execution    |
| `PROCESSING` | Lambda is actively compressing the document          |
| `DONE`       | Processing complete — file is ready to download      |
| `FAILED`     | Processing failed — check `errorMsg` for details     |

## Check job status

Send a GET request to `/api/status/:jobId`:

```bash
curl https://your-api.com/api/status/abc-123 \
  -H "x-api-key: your-api-key"
```

### Response

```json
{
  "jobId": "abc-123",
  "status": "DONE",
  "optimizationLevel": "medium",
  "createdAt": "2026-03-07T10:00:00.000Z",
  "updatedAt": "2026-03-07T10:02:15.000Z",
  "errorMsg": null
}
```

<Info>
  The `updatedAt` timestamp changes each time the job status changes, making it
  useful for tracking processing duration.
</Info>

<Note>
  The status endpoint response is **not** wrapped in the standard `{ success, data }` envelope — the job fields are returned directly.
</Note>

## Polling for completion

Since processing is asynchronous, poll the endpoint at regular intervals until the status is `DONE` or `FAILED`. A **2–3 second interval** is recommended to avoid hitting rate limits.

### JavaScript example

```javascript
const API_BASE = "https://your-api.com/api";
const API_KEY = "your-api-key";

async function waitForCompletion(jobId, intervalMs = 2500, timeoutMs = 60000) {
  const start = Date.now();

  while (true) {
    const response = await fetch(`${API_BASE}/status/${jobId}`, {
      headers: { "x-api-key": API_KEY },
    });

    if (!response.ok) {
      throw new Error(`Failed to check status: ${response.status}`);
    }

    const job = await response.json();

    if (job.status === "DONE") {
      console.log("Processing complete!");
      return job;
    }

    if (job.status === "FAILED") {
      throw new Error(`Processing failed: ${job.errorMsg}`);
    }

    if (Date.now() - start >= timeoutMs) {
      throw new Error("Timed out waiting for job completion");
    }

    await new Promise((resolve) => setTimeout(resolve, intervalMs));
  }
}

// Usage
try {
  const job = await waitForCompletion("abc-123");
  console.log("Completed at:", job.updatedAt);
} catch (error) {
  console.error("Error:", error.message);
}
```

### Python example

```python
import requests
import time

API_BASE = 'https://your-api.com/api'
API_KEY = 'your-api-key'

def wait_for_completion(job_id, interval_seconds=2.5, timeout_seconds=60):
    start = time.time()

    while True:
        response = requests.get(
            f'{API_BASE}/status/{job_id}',
            headers={'x-api-key': API_KEY}
        )
        response.raise_for_status()
        job = response.json()

        if job['status'] == 'DONE':
            print('Processing complete!')
            return job

        if job['status'] == 'FAILED':
            raise Exception(f"Processing failed: {job['errorMsg']}")

        if time.time() - start >= timeout_seconds:
            raise Exception('Timed out waiting for job completion')

        time.sleep(interval_seconds)

# Usage
try:
    job = wait_for_completion('abc-123')
    print(f"Completed at: {job['updatedAt']}")
except Exception as e:
    print(f'Error: {e}')
```

## Handling failures

When a job has `status: "FAILED"`, the `errorMsg` field contains a human-readable explanation:

```json
{
  "jobId": "abc-123",
  "status": "FAILED",
  "optimizationLevel": "medium",
  "createdAt": "2026-03-07T10:00:00.000Z",
  "updatedAt": "2026-03-07T10:01:45.000Z",
  "errorMsg": "File extension not supported."
}
```

Common causes of failure:

- **Invalid or corrupted file** — the file at the S3 key could not be read or parsed
- **Unsupported file extension** — the file extension did not match the declared content type
- **Processing service error** — the downstream compression service returned an error
