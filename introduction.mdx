## What is Doc Optimizer Server?

Doc Optimizer Server is a REST API service that helps you optimize and compress documents efficiently. Built on AWS infrastructure, it provides a simple workflow to upload, process, and download optimized documents with configurable compression levels.

The service handles the entire optimization pipeline asynchronously, allowing you to submit documents for processing and retrieve them when ready.

## How it works

The Doc Optimizer Server follows a straightforward four-step process:

<Steps>
  <Step title="Request upload URL">
    Call the `/api/upload-url` endpoint with your document's content type and desired optimization level. You'll receive a pre-signed S3 URL and a job ID.
  </Step>
  
  <Step title="Upload your document">
    Use the pre-signed URL to upload your document directly to S3. No authentication needed for this step.
  </Step>
  
  <Step title="Start processing">
    Trigger the optimization by calling `/api/process` with the job details. The document will be processed asynchronously.
  </Step>
  
  <Step title="Download optimized document">
    Check the job status with `/api/status/:jobId`. When status is `DONE`, call `/api/download/:jobId` to get a download URL for your optimized document.
  </Step>
</Steps>

## Technology stack

The service is built with:

- **Express.js** - Fast, unopinionated web framework
- **AWS S3** - Scalable object storage for documents
- **AWS Lambda** - Serverless document processing
- **AWS DynamoDB** - NoSQL database for job tracking and API key management
- **AWS SDK v3** - Modern AWS service clients

<Note>
  The server can run both locally for development and as a serverless application in production using AWS Lambda.
</Note>
